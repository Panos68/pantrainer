## Discovery Summary

- **Project / Stack:** PanTrainer — Vite + React (local Mac web app), Notion API (mobile), Markdown (Claude interchange)
- **Area:** Full product — greenfield
- **Type:** New feature (entire initial product)
- **User goal:** "I can log my weekly training sessions and use Claude to generate an updated weekly plan based on my actual performance"
- **API surface:** Notion API (read/write session pages), local filesystem (markdown/JSON), Garmin MCP (future v2)

**Entry points:**
- Mac: `localhost` browser app → view week plan, log sessions, export/import
- Phone: Notion pages (generated by Mac app) → log during workout

**Happy path:**
1. App generates weekly Notion pages (Mon–Sun) pre-filled with planned session
2. During workout on phone: open Notion page → log time / avg HR / calories; gym days: adjust achieved weights (pre-filled from planned)
3. HYROX days: take photo of results sheet, note file path
4. End of week on Mac: "Export Week" → generates `week-YYYY-WW.md` with all logs + rolling 4-week history + athlete profile
5. Paste markdown into Claude chat project + attach HYROX photos → Claude returns new plan markdown
6. Paste Claude response into app → "Import Plan" → app updates next week's plan + Notion pages

**Exit state:** Each session resolved as `completed` or `skipped`; next week's plan loaded

**Session data model:**
- All sessions: date, type, duration, avg HR, active calories, total calories, notes, status
- HYROX days: + photo file paths
- Gym days (Wed + Sun): + exercise list with planned AND achieved sets/reps/weight

**Error states:**
- Session not logged → marked `skipped` with optional note
- Notion sync fails → app works offline, syncs on reconnect
- Claude returns malformed plan → show raw text + manual override
- HYROX photo → attached to Claude chat manually (no in-app OCR)

**Anti-requirements:**
- No native iOS app
- No paid hosting or services beyond existing Claude.ai subscription
- No automated Claude API calls (interaction is manual chat)
- No Garmin integration in v1 (manual RHR/sleep fields, MCP deferred)
- No automated test suite (personal tool)

**Tracking state:**
- Week A/B gym alternation tracked automatically by app
- Deload week: manually flagged by user before export
- RHR/sleep: manual morning check-in (Mac app + optionally Notion)
- Deload counter: app tracks weeks since last deload, warns at week 4, prompts at week 5

**Additional features confirmed in discovery:**
- Rolling 4-week export (not just last week) for better Claude context
- Morning check-in (RHR + sleep) — daily Notion entry or Mac app entry, auto-included in export
- Progress charts: lift trends, HYROX HR trend
- Deload counter with auto-suggestion
- Garmin MCP: build but expect failure; ship as v2 when 429 resolved

**Test focus:** Import/export contract integrity — the markdown format is the critical interface
